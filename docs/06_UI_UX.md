# CatDex UI / UX Design System v1.0

## Purpose

This document defines the complete visual language of CatDex.

Every screen, button, animation and interaction must follow these rules.

The objective is to create a premium mobile game experience.

---

# Design Philosophy

CatDex should feel:

Cute

Modern

Relaxing

Premium

Highly polished

Never look like:

* Dashboard
* Business app
* Productivity app

---

# Inspirations

Animal Crossing

Pokémon GO

Monopoly GO

Marvel Snap

Brawl Stars

Duolingo

The app must have its own identity while borrowing the clarity and polish of these games.

---

# Visual Style

Rounded corners

Soft shadows

Large cards

Bright colors

Friendly illustrations

Lots of white space

Smooth animations

Everything should feel alive.

---

# Cards

Every card has:

Rounded corners

Gradient border (if rare)

Soft shadow

Large illustration

Species name

Variant

Friendship Level

Location

Rarity Badge

Cards should feel collectible.

---

# Animations

Duration

150ms

250ms

500ms

Never exceed one second except special events.

---

# Micro Interactions

Buttons slightly scale down when pressed.

Cards bounce slightly.

XP flies toward the XP bar.

Badges sparkle.

Shiny discoveries emit particles.

Everything feels responsive.

---

# Loading Screens

Never show only a spinner.

Show:

Mascot animation

Funny tips

Discovery facts

Progress indicator

---

# Empty States

Every empty page should encourage exploration.

Example:

"No cats discovered yet.

Let's find your first companion."

# Dark Mode

Dark mode is mandatory.

Not inverted.

Specifically designed.

---

# Motion

Motion must always support usability.

Animations should reward the player, never slow them down.

---

# Sound

Soft UI sounds.

Discovery fanfare.

Rare discovery sound.

Achievement sound.

No annoying repetitive sounds.

---

# Haptics

Use vibration carefully.

Discovery

Achievement

Shiny

Level Up

Premium Purchase

---

# UX Principles

Every tap should give feedback.

Every reward should feel meaningful.

Every screen should encourage another action.

Every session should end with a reason to return tomorrow.
